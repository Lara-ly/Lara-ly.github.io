Live coding: Escribir código fuente para generar música y sonido.

Inicio

Se puede definir el live coding dentro del campo sonoro como una práctica artística hecha con ordenadores que consiste en escribir y modificar código fuente durante un concierto o ensayo para generar música o sonido.
El termino comenzó a utilizarse aproximadamente desde el año 2003, para describir una forma de creación artística con ordenadores en que algunos desarrolladores y artistas escribían y modificaban programas en vivo para generar sonido o imagen frente a un público, mientras que mostraban el proceso de escritura de código al proyectar la pantalla de su ordenador. Desde entonces, el live coding se ha ido conformando como una práctica artística en que se recurre a la escritura de código fuente y su ejecución para expresar música o imagen, además de otras disciplinas como la danza, o la literatura. Esta práctica ha estado ligada a la academia desde su inicio, por lo que se ha escrito bastante acerca de esta.

Se han realizado varios encuentros de live coding, como el International Conference on Live Coding, live.code.fest o *vivo*. A estos encuentros asisten artistas, desarrolladores y teóricos de varios países para intercambiar ideas, escuchar conciertos, presentaciones académicas y ver los últimos desarrollos tecnológicos. Una organización importante para el live coding es la organización TOPLAP, formada en el año 2004, por un grupo de live coders con la intención de explorar, definir y promover esta práctica. Una de las aportaciones de esta organización es un manifiesto sobre live coding, el cual sigue siendo un referente y expresa en uno de sus puntos lo que quizás es la frase más difundida y una contribución a los conciertos de música con ordenadores: "Muéstrenos sus pantallas" o "show us your screens". Con esto se refieren a proyectar el proceso de escritura de código para la audiencia en el momento de un concierto con el objetivo de diluir los misterios de la creación con ordenadores, que realizan programadores, artistas y live coders cuando suben a un escenario para expresarse.

El término live coder se emplea para nombrar al practicante de live coding, quien proviene de distintos campos y no solo de la ciencia de la computación o la música. Es decir, cualquier persona interesada en crear música mediante la escritura de código fuente puede aprender esta práctica y considerarse un live coder. Este punto es importante, ya que las diferentes escenas y comunidades alrededor de esta práctica promueven los valores de diversidad e inclusión. Además de esto, el live coding puede realizarse de manera individual o en conjunto, lo cual permite realizar diferentes formas de colaboración en la creación musical.

El proceso para realizar live coding consiste en escribir código fuente y declarar cada modificación que se realiza. Para ello hay algunos programas especializados como SuperCollider, TidalCycles, Sonic Pi, Extempore o Pure Data, los cuales tienen tres características para lograrlo. La primera es que la forma de traducir el código fuente para su ejecución es interpretada, es decir, cada modificación sobre el código se puede ejecutar de manera independiente sin necesidad de parar el programa para compilarlo. Esto permite modificar el código durante una presentación en vivo de una manera continua. Otra característica es que generalmente estos programas son de alto nivel, es decir, su sintaxis suele ser sencilla, por lo que facilitan la escritura de código para generar sonido en una situación de concierto y por último, la forma de estructurar el código en estos programas es mediante la escritura de patrones o rutinas, lo cual comparte con la música la organización de eventos en el tiempo.
En resumen, un programa interpretado, de alto nivel y que estructura el código en patrones es ideal para realizar live coding musical, sin dejar de mencionar que se requiere un generador de sonido. Por ejemplo, SuperCollider suele generar sonido a través de un servidor llamado scsynth. Mientras que la parte de escritura de código fuente se realiza en su lenguaje sclang.

Cabe mencionar que para generar sonido es posible realizar síntesis o utilizar muestras de audio. De esta manera, podemos pensar el live coding como la organización de eventos sonoros en el tiempo mediante la escritura de instrucciones, patrones y rutinas en diversos lenguajes de programación durante un concierto o ensayo.
